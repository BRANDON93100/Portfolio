🌐 Routage WAN & OSPF Multi-sites

📌 Contexte du Projet

Mise en place d'une infrastructure robuste reliant trois sites européens (Dublin, Londres, Amsterdam). L'enjeu était d'automatiser le routage entre les sites via des liaisons WAN et de centraliser la distribution d'adresses IP pour simplifier l'administration.

🎯 Technologies Déployées

  • OSPF Area 0 : Routage dynamique à état de lien pour une convergence rapide du réseau.
  • DHCP Centralisé : Serveur DHCP situé à Dublin desservant les sites distants.
  • IP Helper-Address : Configuration de relais DHCP sur les routeurs satellites.
  • Liaisons Séries : Simulation de connexions WAN point-à-point entre routeurs.

💻 Configuration Cisco (Dublin & Londres)

! --- DUBLIN (Serveur DHCP & OSPF) ---
ip dhcp pool LAN_LONDRES
 network 172.16.1.0 255.255.255.0
 default-router 172.16.1.1
router ospf 1
 network 172.16.2.0 0.0.0.255 area 0

! --- LONDRES (Relais DHCP) ---
interface GigabitEthernet0/0
 ip helper-address 172.16.2.1

✅ Analyse et Bilan

Grâce à OSPF, le réseau est capable de recalculer instantanément une route si un lien WAN est coupé. L'utilisation du Relais DHCP a prouvé son efficacité en entreprise : elle permet de centraliser la gestion des IP sur un seul serveur tout en desservant des centaines de clients sur des sites géographiquement différents.
