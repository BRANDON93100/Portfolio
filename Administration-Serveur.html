🛠️ Dépannage WordPress (OVH / phpMyAdmin / Cyberduck)

WordPress MySQL / phpMyAdmin OVH Cyberduck (SFTP/FTP)

J’ai résolu un problème de connexion sur un site WordPress hébergé chez OVH. L’intervention a consisté à vérifier la configuration serveur, les accès base de données, puis à mettre à jour les identifiants (wp-config.php) pour rétablir le fonctionnement du site.

📌 Contexte

Le site WordPress n’arrivait plus à se connecter à la base de données (erreur de connexion). L’objectif était d’identifier la cause (identifiants, droits, base, configuration) et de remettre le service en ligne rapidement.

🎯 Objectifs

  • Rétablir l’accès au site WordPress (service web)
  • Vérifier la configuration serveur et les accès à la base de données
  • Mettre à jour de façon sûre les identifiants dans le fichier wp-config.php
  • Valider le correctif par des tests de fonctionnement

🧩 Étapes réalisées (procédure)

  1. Constat de l’erreur WordPress (connexion BDD indisponible).
  2. Connexion à l’hébergement OVH + accès à phpMyAdmin pour contrôler la base et l’utilisateur.
  3. Vérification / réinitialisation du mot de passe MySQL si nécessaire.
  4. Connexion via Cyberduck (FTP/SFTP) pour accéder aux fichiers WordPress.
  5. Mise à jour de wp-config.php (DB_NAME, DB_USER, DB_PASSWORD, DB_HOST).
  6. Test complet : chargement du site + navigation + formulaire / pages.

✅ Résultats

Le site a retrouvé un fonctionnement normal après mise à jour de la configuration. La base de données est de nouveau accessible et l’erreur de connexion a disparu.

🎓 Compétences mobilisées

  • Administration serveur : dépannage d’un service web hébergé
  • Supervision / diagnostic : analyse d’une panne et validation
  • Données (secondaire) : contrôle des accès MySQL via phpMyAdmin
  • Sécurité (bonne pratique) : modification contrôlée d’identifiants + tests
