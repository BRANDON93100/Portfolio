⚙️ Serveur DHCP & DNS (Bind9 / ISC-DHCP)

📌 Pourquoi ce projet ?

Pour qu'une entreprise fonctionne, chaque ordinateur a besoin d'une adresse IP (DHCP) et doit pouvoir traduire les noms des serveurs en chiffres (DNS). J'ai remplacé les fonctions basiques par un serveur Linux professionnel hautement disponible et paramétrable.

🎯 Les piliers techniques

  • DNS (Bind9) : Création de zones pour que les noms de domaine internes renvoient vers les bons serveurs.
  • DHCP (ISC-DHCP-Server) : Attribution automatique et intelligente des adresses IP sur le réseau local.
  • IP Statiques : Réservation d'adresses pour les imprimantes et serveurs via leurs adresses physiques (MAC).

🛠️ Ce que j'ai réalisé

Configuration Bind9

Paramétrage des zones directes et inverses pour une résolution de noms fluide.

Couplage DDNS

Mise à jour dynamique : le DNS s'actualise seul dès qu'une machine reçoit une IP.

✅ Compétences validées

J'ai appris à administrer des services critiques sous Linux Debian. Ce projet m'a permis de comprendre comment les machines communiquent entre elles à bas niveau et comment structurer un réseau d'entreprise robuste.

🖼️ Composants du système

Debian 12

Système d'exploitation stable pour héberger les services réseau.

Zones DNS

Gestion complète des enregistrements A, CNAME et MX.

Pool DHCP

Étendue d'adresses dynamique pour les postes de travail.
