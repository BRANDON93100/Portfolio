🔐 Projet BTS – Sécurisation d’un formulaire de contact

📌 Contexte

Dans le cadre de mon portfolio personnel en BTS SIO option SISR, j’ai développé un formulaire de contact sécurisé afin d’éviter les envois automatisés (spam).

🎯 Objectifs

  • Empêcher les envois automatiques
  • Sécuriser un service web
  • Améliorer l’expérience utilisateur
  • Tester des scénarios de bots

🛠️ Solutions mises en œuvre

  • Champ honeypot invisible (anti-bot)
  • Délai minimal avant l’envoi
  • Blocage automatique des soumissions suspectes
  • Messages utilisateur animés
  • Tests via l’inspecteur du navigateur

🧪 Tests réalisés

  • Soumission trop rapide
  • Remplissage du champ honeypot
  • Simulation de bot via l’inspecteur
  • Envoi légitime après délai

✅ Résultats

Le formulaire est fonctionnel pour un utilisateur réel et bloque efficacement les tentatives automatisées, sans impact négatif sur l’expérience utilisateur.

🎓 Compétences BTS mobilisées

  • Sécurisation des services
  • Analyse des risques
  • Tests et validation
  • Cybersécurité applicative

🖼️ Preuves et captures – Tests anti-spam

Afin de prouver le bon fonctionnement du système anti-spam, des tests ont été réalisés manuellement à l’aide de l’inspecteur du navigateur (F12). Les captures ci-dessous montrent les différentes étapes du test.

Présence du champ honeypot invisible dans le code HTML, non visible pour l’utilisateur.

Simulation d’un bot en remplissant manuellement le champ honeypot via l’inspecteur du navigateur.

Blocage automatique de l’envoi avec affichage du message indiquant une tentative de spam.

Conclusion : le système anti-spam fonctionne correctement et bloque les tentatives automatisées tout en laissant passer les envois légitimes.
